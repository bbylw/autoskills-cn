为什么需要 autoskills

AI 助手很强大，但它不了解你的项目

你的 AI 编码助手可以写代码，但它不知道 React 19 的新特性、Next.js App Router 的约定、或者 Prisma 的查询最佳实践。每次对话，你都在重复解释相同的上下文。

没有 autoskills
你: 帮我用 React 写一个表单组件
AI: 好的，这是一个使用 useState 的基本表单组件……(通用回答，不考虑你的项目规范)
有了 autoskills
你: 帮我用 React 写一个表单组件
AI: 根据项目规范，推荐使用 React Hook Form + Zod 验证，搭配 shadcn/ui 的 Form 组件……(针对你的技术栈)

工作原理

四步完成，零配置

从运行命令到技能就绪，整个过程完全自动化。

01

运行命令

在项目根目录执行 npx autoskills

02

智能检测

扫描 package.json、Gradle 文件及各类配置文件

03

安装技能

从 skills.sh 匹配并安装最佳 AI 技能

04

即刻生效

AI 助手自动获得项目专属知识，无需额外操作

Claude Code 集成

为 Claude Code
深度集成

当自动检测到 Claude Code 或通过 -a 参数指定时，autoskills 会在项目根目录生成 CLAUDE.md 摘要文件。

  • → 自动检测 Claude Code 环境
  • → 汇总所有已安装的 Markdown 技能文件
  • → 生成结构化的技能摘要
  • → Claude 即刻获得项目技能上下文
◆ CLAUDE.md
# 项目 AI 技能摘要
## 已安装技能
- react-best-practices.md
- nextjs-app-router.md
- typescript-patterns.md
- tailwind-utilities.md
## 检测到的技术栈
React 19, Next.js 15, TypeScript 5.7, Tailwind CSS 4
## 使用提示
请参考上述技能文件中的最佳实践
进行代码编写和架构设计。

命令选项

灵活可控

-y, --yes

跳过确认

跳过安装确认提示，直接执行。适合 CI/CD 或批量操作。

--dry-run

预览模式

仅展示将要安装的技能，不实际安装。安装前确认匹配结果。

-h, --help

帮助信息

显示完整的帮助文档和使用说明。
